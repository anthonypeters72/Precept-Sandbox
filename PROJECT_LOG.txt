[2026-03-09]
Major Milestone

Established a reusable corpus ingestion pipeline and successfully integrated The Testaments of the Twelve Patriarchs into the Precept engine. This marks the first full demonstration of the system’s ability to take a raw text source and transform it into a structured corpus usable by the engine for verse lookup, cross-corpus similarity, and precept discovery.

Work Completed

1. Corpus Parsing Pipeline

Created a custom parser (parse_patriarchs.py) to extract structured verse data from the Patriarchs PDF.

Implemented logic to handle:

chapter markers

verse markers

paired verses (e.g., “1,2”)

inline verse numbering irregularities.

2. Corpus Validation Tool

Built validate_corpus.py to ensure corpus integrity.

Validation checks include:

duplicate verse keys

missing text rows

invalid chapter/verse values

ordering regressions

verse gap detection.

Final validation results:

Unique verse keys: 782
Duplicate rows: 0
Empty text rows: 0
Bad chapter/verse rows: 0
Verse gaps found: 7
Ordering regressions: 0

Remaining verse gaps are due to formatting irregularities in the source text and are considered acceptable for current engine usage.

3. Corpus Integration

Added Data/patriarchs/patriarchs.csv.

Implemented a structured CSV loader within engine.py.

Registered the corpus within the engine’s corpus registry.

Updated lookup_text_any() to include the Patriarchs corpus.

Added book normalization aliases for the twelve Testament books.

4. Engine Verification

Confirmed successful verse resolution:

e.g. Testament of Judah 3:1

Confirmed cross-corpus precepts function correctly:

Patriarchs ↔ Jasher

Patriarchs ↔ KJV

Verified fallback and near-miss ranking logic remains stable.

Example output showed relevant matches involving:

Jasher

Joshua (KJV)

internal Patriarch passages.

Architecture Achieved

The project now supports a full corpus ingestion workflow:

PDF / Source Text
      ↓
Parser Script
      ↓
Structured CSV (book,chapter,verse,text)
      ↓
Corpus Validator
      ↓
Data/<corpus> directory
      ↓
Engine Loader
      ↓
FastAPI Endpoint
      ↓
Precept Search + Cross-Corpus Linking

This pipeline allows additional corpora to be added with minimal manual effort.
